您可以使用 Postman：

mac平台值得使用的软件推荐

发表于 2015-04-16   |   分类于 软件

从接触并使用mac已经有一年了，在以前使用windows时就喜欢折腾软件，转移到mac平台后，仍然喜欢去找一些用起来顺手的软件，现在把我认为不错的几款软件罗列出来分享给大家。

Spotlight

Spotlight是mac os上自带的搜索+快速启动工具。更新到mac os 10.10之后，Spotlight 拥有了一个全新的搜索框界面，你不仅可以使用 Spotlight 来快速启动APP，还可以搜索本地文件、联系人、日历、邮件、信息、词典、地图、系统设置等内容，而且现在还能搜索到 App Store 和 iTunes Store 上的全部应用和音乐电影等内容。当然，它还可以搜索互联网(维基百科、Bing等等)。你可以为Spotlight设置相应的快捷键，来获得更好的使用体验。如果你觉得Spotlight的功能还满足不了你的胃口，你可以试试Flashlight这款插件.

Flashlight一款由 Github 用户 Nate Parrott 发布，众多开发者参与的 Spotlight 插件,它支持Baidu Search（百度搜索）、Taobao Search（淘宝搜索）、V2EX、知乎日报和豆瓣电影等等快捷功能方式，你可以在插件库中安装你需要的插件。你甚至可以直接在Spotlight中输入shutdown来进行关机操作。

Alternote

Evernote是我最常用的笔记软件了，几乎每天都要使用，但是Evernote的反应速度比较慢，功能繁多，许多功能都用不到，经过一番寻找和试用，发现了Alternote这款第三方的Evernote客户端，安装包体积才4M大小，拥有Evernote的基础功能，界面简洁流畅，非常符合我的审美，经过一段时间的使用，觉得相当不错，推荐给大家。

Atom

Atom 是 Github 专门为程序员推出的一个跨平台文本编辑器。具有简洁和直观的图形用户界面，并有很多有趣的特点：支持CSS，HTML，JavaScript等网页编程语言。它支持宏，自动完成分屏功能，集成了文件管理器。最近，Atom更新到了0.177版本，我也特地下载尝试了一下。
让我惊奇的是，Atom底层架构竟然是基于Chromium的，所有的窗口都是一个本地渲染的网页。难怪我用起来有几分浏览器的影子，使用起来不像原声软件那么流畅。Atom 之前是采用了 node.js 来访问文件系统和包管理。这样就让Atom的包管理具有很强的灵活性和可扩展性。面对浩如烟好的npm资源，Atom 的可配置型也变得异常突出。本次发布的0.177.0版，其中一个引入注目的变化是从 Node.js 切换到了 io.js。io.js是Node.js的分支，Node.js社区发生分裂后由核心开发者在去年12月创建的，已经发布了v1.1版，目前开发非常活跃。Atom是切换到io.js的一个重量级项目。

得益于nodejs的扩展性，Atom自带70多种插件，包含了常用的功能，这个项目最近也比较火热，相信不断会有更好的插件推出。

iBook

iBook是mac os自带的图书阅读工具，提供书籍管理，同步，笔记以及优秀的排版功能。支持epub,pdf等格式。

ios开发中的GET和POST方法对比

发表于 2015-04-04   |   分类于 iOS开发

在iOS开发中会经常用到GET和POST这两种数据请求方法，说起来也很简单，但是这两者的区别还是需要注意的。

  • GET - 从指定的资源请求数据。
  • POST - 向指定的资源提交要被处理的数据

##GET 方法

有关 GET 请求的一些注释：

  • GET 请求可被缓存
  • GET 请求保留在浏览器历史记录中
  • GET 请求可被收藏为书签
  • GET 请求不应在处理敏感数据时使用
  • GET 请求有长度限制，地址最多255字节，将参数直接写在访问路径上。操作简单，不过容易被外界看到，安全性不高
  • GET 请求只应当用于取回数据

##POST 方法

有关 POST 请求的一些注释：

  • POST 请求不会被缓存
  • POST 请求不会保留在浏览器历史记录中
  • POST 不能被收藏为书签
  • POST 请求对数据长度没有要求，将参数放到body里面。POST请求操作相对复杂，需要将参数和地址分开，不过安全性高，参数放在body里面，不易被捕获

开始写博客了

发表于 2015-03-18   |   分类于 随笔

最近在36kr看了一篇文章，“为什么软件工程师应该养成写作的习惯？”，觉得说的很有道理。的确，作为一名刚入行一年的“码农”，很有必要锻炼自己的逻辑思考能力，写代码和写作一样，都是对大脑的一种锻炼。写文章和写代码的最核心的共同支出在它们都需要清晰地思考能力，软件工程师应该写作因为现在开发协作变得越来越重要，即便没人阅读你的文章，协作的过程也是有益无害。

工作一年了，我也确实认识到自己在逻辑思考能力上的不足，注意力不集中，有时候很难静下心来思考一些事情。经常会对一些突发事情不能应对自如。而写博客既能记录日常生活中的点滴，又能把工作中的经验记录下来与人分享，还能锻炼自己的逻辑思维能力，真的是再好不过了。

难得最近比较清闲，抽出来半天时间，在github pages上建立了静态blog，学习安装了hexo,fork了janman项目，简单修改了下，在本地markdown了这篇文章，commit，大功告成！

感谢github提供了这么好的服务，这一切都是免费的！
