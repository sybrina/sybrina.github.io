Keychain error -34018

发表于 2016-01-12   |   分类于 iOS开发

在iOS开发中，我们通常会使用NSUserDefault来存储一些简单的数据，NSUserDefault使用起来也比较简单，但是对于比较敏感的数据，比如用户名、密码、银行账号等数据，使用iOS自带的keychain来进行存储会比较靠谱。

原来是iOS的bug

直接使用系统提供的keychain会比较复杂，而且需要自己处理一些错误情况，所以为了更方便的使用keychain，我在github找到了一款开源的sskeychain用来操作keychain。sskeychain提供的API比较简单，使用起来顺手，但是用了一段时间后发现keychain经常会报错 “OSStatus error: [-34018] Security error has occurred.” ，开始还以为是sskeychain的问题，使用了UICKeyChainStore来替换sskeychain，结果还是出现这个错误，Google后发现这原来是iOS的一个bug，直到现在还没有修复。关于bug的讨论在这里：Github、Apple forums。有开发者在使用 DropboxSDK 时也遇到了这个bug，显示这个bug从 iOS 8开始就有了。

何时发生

讨论中提到这个bug经常会在 “Received memory warning.” 或者”application:didFinishLaunchingWithOptions and applicationDidBecomeActive:“调用后出现，也就是系统收到内存警告或者application:didFinishLaunchingWithOptions和applicationDidBecomeActive尚未执行完毕就访问keychain就会出现错误。

对于这个bug，暂时也没有找到好的解决方案，如果你有好的办法，欢迎和我进行探讨。

期待Apple能早日修复这个bug！
